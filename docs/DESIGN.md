# ThyroPADA设计基线

版本：v0.2  
日期：2026-05-09  
门控：L2 轻量审查与既有页面视觉迭代

## 设计目标

ThyroPADA 用于医疗/科研场景中的甲状腺超声图像判读、图像集维护和结果统计。界面应当像真实科室工具，而不是营销页或 AI 生成模板：克制、清晰、可信、可长期使用。

## 本次审计结论

原界面主要问题：

- 深色玻璃、光晕、蓝紫径向背景和渐变按钮带来明显模板感。
- 英文装饰性 eyebrow 与医疗中文工作流不匹配。
- 面板和卡片过度依赖半透明与大圆角，真实产品密度不足。
- 结果页使用渐变分数、逐题行左侧彩色边框，视觉表达偏装饰。
- 备注输入在判读页只监听第一次输入，连续编辑存在保存风险。

## 设计方向

方向名称：临床科研工作台

视觉关键词：

- 浅色底
- 低饱和青绿
- 明确边框
- 表格化信息
- 真实状态反馈

界面原则：

- 不使用蓝紫渐变、光晕、玻璃拟态、装饰性圆球。
- 优先让图片、题号、保存状态、结果统计成为焦点。
- 操作控件用实体按钮和清晰边框表达层级。
- 重复信息用表格与紧凑卡片承载，不做营销式大留白。

## Token

颜色：

- 背景：`#f5f7f4`
- 页面弱背景：`#eef3ef`
- 表面：`#ffffff`
- 主文字：`#17211f`
- 次文字：`#485652`
- 辅助文字：`#6a7773`
- 主色：`#0f6f68`
- 主色深色：`#0a514d`
- 成功：`#257a54`
- 警示：`#8b6b1f`
- 危险：`#ad3f3a`
- 边框：`#d7e0dc`

排版：

- 字体：`Aptos`、`Segoe UI`、`PingFang SC`、`Microsoft YaHei`、系统无衬线。
- 正文基准：15px / 1.55。
- 标题不使用负字距，不使用渐变文字。
- 数字与统计值使用 `tabular-nums`。

形态：

- 小控件圆角：4px。
- 面板/重复项圆角：8px。
- 大容器上限圆角：12px。
- 阴影仅用于悬停或浮层，不作为主要装饰。

## 页面规范

首页：

- 首屏是产品与用户状态，不做 hero 营销结构。
- 任务以紧凑任务卡呈现，强调任务名称、发布状态、答案选项和开始动作。

判读页：

- 左侧进度固定在桌面端，移动端自然堆叠。
- 题号导航使用当前/已评估/未答三态，不只依赖颜色。
- 主图区域保留深色底以适配超声图像边缘。
- 底部操作区可粘住视口，减少翻题和保存的鼠标移动成本。

后台：

- 表格优先，强调可扫描性。
- 筛选器与导出保持工具化，不做大卡片包装。

结果页：

- 分数使用纯色数字，不使用渐变。
- 逐题状态通过 chip 和轻微背景差异表达，避免模板化左边框。

## 交互状态

所有按钮、链接、输入、选择器需要具备：

- hover
- active
- focus-visible
- disabled
- loading/保存中状态（由现有 JS 文案承载）

无障碍约束：

- 触摸目标不小于 40px，关键判读选项不小于 44px。
- 表格容器允许横向滚动，避免移动端截断。
- 尊重 `prefers-reduced-motion`。

## 质量评分

变更前：C-  
主要原因：功能完整，但视觉明显偏 AI 模板，医疗可信感不足。

变更后目标：B+  
通过标准：去掉蓝紫玻璃感，形成一致的医疗科研工作台视觉，并保留清晰的判读效率。
